## Описание
Данный проект определяет пользовательский интерфейс, связывая конечного пользователя с системой поддержки принятия врачебных решений.

## Струткура проекта

```
client_side/
├── src/
│   ├── assets/
│   │   └── style.css          # Стилизация страницы
│   ├── components/
│   │   ├── CreateDoc.vue      # Раздел добавления документов
│   │   └── DocList.vue        # Раздел со списком имеющихся документов
│   ├── App.vue                # Навигация и раздел с чатом
│   └── main.js                # Подключение и монтирование App.vue
└── index.html                 # Основа страницы
```
### Вспомогательные файлы
`Dockerfile, .dockerignore, compose.yaml, nginx.conf` - для сборки Docker image.
`babel.config.js, jsconfig.json, vue.config.js` - для сборки Vue приложения.
`package-lock.json, package.json` - зависимости Vue приложения.

## Запуск и настройка

### Локальная разработка
1. Установите зависимости:
   ```bash
   npm install
   ```
2. Запустите бэкенд (`uvicorn` или `docker compose` в корневой директории проекта).
3. Запустите фронтенд:
   ```bash
   npm run serve
   ```
   Приложение откроется на `http://localhost:8080`. WebSocket подключится к чату автоматически, используя тот же хост (`http://<текущий hostname>:8002`). При необходимости можно задать `VUE_APP_API_BASE_URL`, если бэкенд находится на другом сервере.

### Docker
_Требуется Docker Desktop или совместимый движок._

1. Запустите контейнеры из корня проекта:
   ```bash
   docker compose up --build
   ```
   Будут подняты сервисы `db`, `app` (бэкенд) и `frontend`.
2. Страница доступна на `http://localhost:8080`. Браузер обращается к бэкенду по тому же хосту (`http://<host>:8002`) — никаких дополнительных переменных окружения задавать не нужно. Если бэкенд расположен отдельно, можно задать `VUE_APP_API_BASE_URL` перед сборкой.

### Основные функции
- **Чат** — интерактивная лента сообщений, статус соединения, автоматическое переподключение.
- **Список документов** — пагинация (по 50 записей), поиск по названию или ID, открытие PDF в новой вкладке, удаление.
- **Добавление документа** — загрузка PDF с метаданными через `/createdoc`, сообщение об успехе/ошибке.

При отсутствии переменной окружения `VUE_APP_API_BASE_URL` фронтенд пытается подключиться к `http://<hostname>:8002`.
